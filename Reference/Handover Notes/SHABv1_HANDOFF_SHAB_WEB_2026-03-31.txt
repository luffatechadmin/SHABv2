SHABv2 / SHAB Dashboard (React + Vite) – Handoff Notes (Updated 2026-04-16)

Repository root:
  C:\Users\<you>\Documents\GitHub\SHABv2

Web app folder:
  SHAB Dashboard\

Tech stack:
  - Vite (dev server: http://localhost:5173/)
  - React 18
  - TypeScript
  - TailwindCSS
  - Supabase JS client (@supabase/supabase-js)


1) Key files and where to start

Entry points:
  SHAB Dashboard\index.html
  SHAB Dashboard\src\main.tsx
  SHAB Dashboard\src\App.tsx

Supabase client:
  SHAB Dashboard\src\lib\supabase.ts
    - Uses VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY (publishable key).


2) Install / run commands (SHAB Dashboard)

Install dependencies:
  cd "C:\Users\<you>\Documents\GitHub\SHABv2\SHAB Dashboard"
  npm ci

Create local env file:
  SHAB Dashboard\.env.local
  Use SHAB Dashboard\.env.example as template:
    VITE_SUPABASE_URL=...
    VITE_SUPABASE_ANON_KEY=...

Run dev server:
  npm run dev

Build:
  npm run build


3) Environment variables (frontend)

The frontend reads:
  VITE_SUPABASE_URL
  VITE_SUPABASE_ANON_KEY

Template:
  SHAB Dashboard\.env.example

Recommended local file:
  SHAB Dashboard\.env.local
    - Do not commit this file.


4) Relation to SHAB Attendance System (WL10Middleware)

Attendance ingestion should be handled by the middleware:
  SHAB Attendance System\SHAB Attendance Middleware

The middleware writes to Supabase using the service role key:
  SUPABASE_SERVICE_ROLE_KEY

Important:
  - Do not ship service role key to the browser.
  - Only store SUPABASE_SERVICE_ROLE_KEY in server-side env files or in the middleware dashboard settings.
