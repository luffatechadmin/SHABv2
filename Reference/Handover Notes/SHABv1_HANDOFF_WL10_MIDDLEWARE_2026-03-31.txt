SHABv2 / WL10 Middleware – Handoff Notes (Updated 2026-04-16)

Repository root:
  C:\Users\<you>\Documents\GitHub\SHABv2

Key project involved:
  SHAB Attendance System\SHAB Attendance Middleware

Client deliverable folder:
  SHAB Attendance System\Client Package


1) Recent key changes (high level)

Dashboard UX:
  - Login page no longer shows default credential hint.
  - After login, dashboard redirects to Settings -> Connection Summary.
  - Device Status is moved under Settings as Connection Summary.
  - Default auto-sync (when no schedule is configured) is 1 hour.

Supabase data model:
  - Employee List uses Supabase table: public.staff
  - Shift Patterns use Supabase table: public.shift_patterns
  - Attendance uses Supabase tables:
      public.attendance_events
      public.attendance_intervals
      public.attendance_records


2) What to run (development)

Build (Release):
  cd "SHAB Attendance System\SHAB Attendance Middleware"
  dotnet build -c Release

Run dashboard (Kestrel):
  dotnet run -c Release -- --dashboard --dashboard-port 5099

Open:
  http://127.0.0.1:5099/

If port 5099 is already in use:
  dotnet run -c Release -- --dashboard --dashboard-port 5100


3) What to run (client PC)

Copy the folder:
  SHAB Attendance System\Client Package

First-time setup:
  - Run Start Dashboard.bat
  - This installs the ZKTeco SDK (UAC required), starts the middleware, opens the dashboard, and creates a Desktop shortcut.

Stop:
  - Run Stop Dashboard.bat


4) Supabase requirements / env vars

Middleware uses privileged Supabase access (server-side only):
  SUPABASE_URL
  SUPABASE_SERVICE_ROLE_KEY

Never commit or share SUPABASE_SERVICE_ROLE_KEY.
Rotate it if it was ever committed to git history.

Schema (run in Supabase SQL Editor for a new project):
  1) SHAB Dashboard\supabase\migrations\20260416000000_create_staff_and_shift_patterns.sql
  2) SHAB Dashboard\supabase\migrations\20260326000000_create_attendance_records.sql


5) Common issues and quick checks

Dashboard refuses to connect:
  - Ensure WL10Middleware.exe is running.
  - Ensure port 5099 is not already in use.

Client PC fails to start middleware:
  - Ensure .NET 8 Runtime (Windows x86) is installed.
  - Ensure ASP.NET Core Runtime 8 (Windows x86) is installed.
  - If using COM mode, ensure ZKTeco zkemkeeper is installed/registered.
