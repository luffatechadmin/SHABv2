Changes made:
  A) Native TCP protocol constant fix
     - Updated ZK TCP “top header” constant to match pyzk:
       MachinePrepareData2 = 32130

  B) Attendance record size parsing (COM path) adjustment
     - When reading the COM-provided attendance blob (ReadWithBuffer/CMD 13), recordSize selection now prefers 16-byte records first:
       if (totalSize % 16 == 0) recordSize = 16;
       else if (totalSize % 40 == 0) recordSize = 40;
       else if (totalSize % 8 == 0) recordSize = 8;

  C) Added file import mode: --import-attlog
     - New CLI option:
         --import-attlog <path>
       This reads a text attendance export (like Reference\1_attlog.txt), converts each row into Punch records, then upserts to Supabase using the existing UpsertToSupabase logic.
     - Supports combining with:
         --clear-attendance   (clears Supabase attendance_events for this device first)
         --verify             (prints latest attendance_events after upsert)
         --dry-run            (prints what would be upserted; no Supabase writes)

  D) Import timestamp interpretation toggle
     - Environment variable:
         WL10_IMPORT_ATTLOG_TIME_KIND
       Values:
         utc   (default)  = treat timestamps in the text file as UTC clock time (no timezone shift)
         local            = treat timestamps in the text file as local machine time, convert to UTC for occurred_at
       Notes:
         - Use utc if the Notepad timestamp should match occurred_at time-of-day.
         - Use local if your text file timestamps are local time and you want correct UTC conversion.

  E) Export Reference\1_attlog.dat (text format; overwrites)
     - During syncing, the middleware exports a text-format file to:
         SHABv1\Reference\1_attlog.dat
     - Output columns match WL10 USB export layout:

     - VerifyMode/“verified” values now match manual USB export.

4) Common pitfalls encountered and how to avoid them

“.\WL10Middleware.exe is not recognized …”
  - You ran the command from a folder that doesn’t contain the exe (e.g., shab-web).
  - Fix: cd into the Release output folder (see commands above) or execute using the full path to the exe.

Foreign key errors when upserting attendance (missing staff rows)
  - Fix: set WL10_AUTO_CREATE_STAFF=1 so middleware creates staff rows before inserting attendance.

Watermark preventing “full sync”
  - The middleware can keep a local watermark state file:
      state.json (in the executable’s base directory)
  - If you want to force a full backfill, use:
      --clear-attendance
    which clears Supabase rows for the device and deletes state.json.
  - Alternatively, set:
      WL10_WATERMARK_SOURCE=none
    to ignore the watermark logic for that run.

Dashboard not reachable from another device on Wi‑Fi
  - Confirm the PC IP with ipconfig and use:
      http://<PC-IP>:5099/
  - Windows Firewall can block inbound connections; allow inbound TCP 5099.
  - Guest Wi‑Fi / “client isolation” can block device-to-PC access.

Device IP reachable from PC (network/VLAN issue)
  - If your PC IPv4 is not on the same subnet as the WL10 IP, you will see:
      Test-NetConnection <WL10_IP> -Port 4370  -> TcpTestSucceeded False
  - Fix by connecting the PC to the correct LAN/VLAN, or enable routing + allow TCP 4370.

  C:\Users\HP\Documents\GitHub\SHABv1\Reference\1_attlog.dat
  C:\Users\HP\Documents\GitHub\SHABv1\Reference\Manual - 1_attlog.dat

Expected line format:
  staffId<TAB or spaces>YYYY-MM-DD HH:MM:SS<TAB>verified<TAB>status<TAB>workcode<TAB>reserved

Example lines:
  9  2026-03-31 06:09:17  1  255  1  0
  8  2026-03-31 17:01:06  1  1    1  0

Importer behavior:
  - Only staff ID and timestamp are required; extra columns are ignored by the importer.
  - Staff IDs are passed through the existing MapStaffId logic (prefix/padding if configured).
  - Event IDs are stable (device_id + staff_id + occurred_at_utc) to dedupe on re-import.
