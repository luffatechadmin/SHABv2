SHABv2 – New PC Setup Guide (Supabase + GitHub + Node + .NET) (Updated 2026-04-16)

Goal:
  Get a new Windows PC ready to continue development and client packaging for:
    - SHAB Dashboard (React/Vite)
    - SHAB Attendance System (WL10Middleware .NET 8 Windows, client package scripts)


1) Accounts and access (do first)

GitHub:
  - Confirm you can access the SHABv2 repository.
  - Preferred: set up SSH keys for GitHub (or use HTTPS + credential manager).

Supabase:
  - Confirm you can access the target Supabase project in the Supabase dashboard.
  - Confirm you can retrieve keys:
      SUPABASE_URL
      SUPABASE_ANON_KEY (publishable key, safe for browser)
      SUPABASE_SERVICE_ROLE_KEY (server-side only)
  - Keep the service role key private:
      - Never commit it to git
      - Never put it into VITE_* variables


2) Software to install on the new PC

Required:
  - Git (latest)
  - Node.js (LTS recommended)
  - .NET SDK 8.x (Windows)

Required for the shipped client package (win-x86):
  - .NET 8 Runtime (Windows x86)
  - ASP.NET Core Runtime 8 (Windows x86)

Recommended:
  - VS Code (or Visual Studio) + C# + TypeScript tooling
  - Windows Terminal / PowerShell

Optional (only needed if you will use the WL10 COM/zkemkeeper SDK path on that PC):
  - ZKTeco zkemkeeper.dll (32-bit) and its dependencies
  - Ability to register COM DLLs using regsvr32


3) Clone the repo

Choose a working folder (example):
  C:\Users\<you>\Documents\GitHub\

Clone:
  git clone <repo-url>
  cd SHABv2


4) SHAB Dashboard setup (frontend)

Folder:
  SHABv2\SHAB Dashboard

Install deps:
  npm ci

Create local env file:
  SHAB Dashboard\.env.local
  Use SHAB Dashboard\.env.example as a template:
    VITE_SUPABASE_URL=...
    VITE_SUPABASE_ANON_KEY=...

Run:
  npm run dev
  (Vite dev server default: http://localhost:5173/)


5) Supabase schema (required tables)

For a new Supabase project, run these SQL files in Supabase SQL Editor:
  1) SHAB Dashboard\supabase\migrations\20260416000000_create_staff_and_shift_patterns.sql
  2) SHAB Dashboard\supabase\migrations\20260326000000_create_attendance_records.sql

Key tables created/used by SHAB Attendance System:
  - public.staff
  - public.shift_patterns
  - public.attendance_events
  - public.attendance_intervals
  - public.attendance_records


6) SHAB Attendance System (WL10Middleware) – development run

Folder:
  SHABv2\SHAB Attendance System\SHAB Attendance Middleware

Build:
  dotnet build -c Release

Run local dashboard (Kestrel):
  dotnet run -c Release -- --dashboard --dashboard-port 5099

Open:
  http://127.0.0.1:5099/

If port 5099 is already in use:
  dotnet run -c Release -- --dashboard --dashboard-port 5100


7) SHAB Attendance System – client package run

Client deliverable folder:
  SHABv2\SHAB Attendance System\Client Package

First-time setup and run:
  - Run Start Dashboard.bat
  - It will handle:
      - ZKTeco SDK install (UAC required)
      - Middleware start and browser open
      - Desktop shortcut creation

Client env file used by the packaged app:
  SHAB Attendance System\Client Package\App\win-x86\.env.local

Required Supabase env vars for the middleware:
  SUPABASE_URL
  SUPABASE_SERVICE_ROLE_KEY

Required Supabase env vars for the browser-only Supabase client (if used):
  VITE_SUPABASE_URL
  VITE_SUPABASE_ANON_KEY


8) Operational notes (avoid common issues)

Keys safety:
  - Never commit SUPABASE_SERVICE_ROLE_KEY.
  - Rotate it if it was ever committed to git history.

Sync interval:
  - Default poll interval when no schedule is configured:
      WL10_POLL_INTERVAL_SECONDS=3600

Port conflicts:
  - If the dashboard refuses to connect, ensure the middleware process is running and the port is free.
