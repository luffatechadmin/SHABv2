4) WL10 poller script (Node)

Scripts (package.json):
  npm run wl10:poll:once
  npm run wl10:poll

Important:
  - The poller uses SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY (service role key).
  - This is privileged. Do not ship service role key to the browser.
  - The poller loads env vars via dotenv/config, so you can place variables in:
      shab-web\.env
    or configure them in your shell session before running.

Poller configuration env vars (optional + defaults from scripts\wl10-poller.mjs):
  WL10_BASE_URL         (default: http://192.168.1.170)
  WL10_LOG_PATH         (default: /)
  WL10_LOG_MODE         (default: auto)
  WL10_USER_ID_FIELD    (default: userId)
  WL10_OCCURRED_AT_FIELD(default: occurredAt)
  WL10_DEVICE_ID        (default: WL10-192.168.1.170)
  WL10_TIMEOUT_MS       (default: 8000)
  WL10_INTERVAL_SECONDS (default: 30)
  WL10_USERNAME         (optional; enables HTTP Basic auth)
  WL10_PASSWORD         (optional; enables HTTP Basic auth)

Poller flags:
  --once
  --dry-run
  --interval <seconds>


5) Common pitfalls

Missing VITE_* env vars:
  - The app will warn if VITE_SUPABASE_URL / VITE_SUPABASE_ANON_KEY are missing.
  - Create shab-web\.env.local based on shab-web\.env.example.

Running WL10Middleware.exe from shab-web folder:
  - If you run PowerShell from shab-web\, you must either cd to the middleware exe folder or use the full path to the exe.


6) Relation to WL10Middleware (.NET)

There are two attendance ingestion paths now:
  A) .NET WL10Middleware (native TCP / COM / file import) -> Supabase
  B) Node poller (wl10-poller.mjs) -> Supabase

If you are using WL10Middleware, you do not need to run the Node poller.
If you are using the Node poller, keep the service role key out of the browser and only run it as a script/server-side process.


7) Recent related progress (WL10Middleware)

As of 2026-04-06, the WL10Middleware (.NET) side now:
  - Exports the attendance file to:
      SHABv1\Reference\1_attlog.dat
    using a text format like:
      staff_id<TAB>YYYY-MM-DD HH:MM:SS<TAB>verified<TAB>status<TAB>workcode<TAB>reserved
  - Updates the WL10Middleware local dashboard Device Records tab to read and display 1_attlog.dat exactly (same columns/order).

If you are only working on shab-web UI, this does not change your build/dev commands, but it affects how attendance data may be produced locally for Supabase during testing.
